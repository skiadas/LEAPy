The my_utils Module¶

The mu_utils module contains certain constants and helper functions, in addition to the Placement class, which performs the actual grunt work of placing students based on a given set of rules.

standardize_key(key)[source]¶

Uses the key associations loaded from key_associations.txt to standardize the value of key. Returns None if it does not find the key. This function is used by the student.Student class to standardize the keys allowed by that class. The associations can be found in the file key_associations.txt. Change the file at your own discretion.

JINJA_TEX_ENV¶

The Jinja2 environment used for LaTeX templates. See Creating Templates for more details.

JINJA_HTML_ENV¶

The Jinja2 environment used for HTML templates. See Creating Templates for more details.

The Placement class¶

class Placement[source]¶

The Placement class forms an umbrella for all operations needed with regards to placement. It reads the information in placement_instructions.txt to produce appropriate Singleton classes for each placement type, as well as the instructions on how to place a student. For more information on this file, see The placement_instructions.txt file.

_placements¶

A dictionary of possible placements. The values are objects of a private class, whose attributes and corresponding values are the pairs specified for that placement in placement_instructions.txt and are read at start-up.

_expressions¶

A list of pairs (expression, placement), where expression is an expression to be evaluated for each student, the placement is the corresponding placement key, from _placements.

classmethod Perform_placement(student)[source]¶

Goes through the list in _expressions, processing those expression for the current student, and returning the placement corresponding to the first True expression.
