The Student Class¶

class Student(*args, **kwargs)[source]¶

Bases: dict

The Student class represents one student. It works as a dictionary but only accepts certain keys and automatically converts certain other keys to them. TODO create list of the keys. You usually don’t need to work with this class directly, use studentList.StudentList instead. Here are the main keys (this can be changed by editing the file key_associations.txt):

Variables:
  • id – The student’s id number.
  • last – Last name
  • first – First name
  • nick – Nickname / Preferred name
  • status – International student, Freshman etc
  • act – ACT score (Math)
  • sat – SAT score (Math)
  • placement_score – The score in the placement exam
  • placement_name – Deprecated. Use placement instead
  • past_courses – A & separated list of abbreviations for courses the student took in High School. Used to determine for instance if the student has taken Calculus.
  • placement – The student’s placement. It is a my_utils.Placement object.
  • major – A | separated list of possible majors the student might be interested in.
  • math_interest – y/n/m (maybe) depending on whether the student is interested in math.
  • know_integrals – y/n depending on whether the student knows about integrals.
  • room – The room the student is assigned for LEAP
  • hc_grade – The student’s average letter grade in High School
get(key, default=None)[source]¶

All getters and setters have been overwriten, in order for key standardization to take place.

update(*args, **kwargs)[source]¶

Replaces keys or adds new keys based on those provided, after standardizing them. This, along with safe_update(), are the recommended method for updating Student objects.

safe_update(*args, **kwargs)[source]¶

Like update() but will not change existing keys. Useful for merging information from new data sources.
