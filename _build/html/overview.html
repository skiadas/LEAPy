General OverviewÂ¶

The functionality of LEAPy revolves around the following classes. Only a brief preview provided here, look at their individual pages for more details.

  • student.Student Corresponds to the record of a single student.
  • studentList.StudentList A list of students read form a spreadsheet/dataset. This is the main object we work with.
  • dataset_utils.Dataset Helper class that allows reading and writing datasets in various formats, CSV, Excel and HTML currently supported.

Typically we use dataset_utils.Dataset.Read() to process a dataset from a file/string, then use StudentList‘s constructor to create a StudentList object. Merge information from another dataset using StudentList.update_from_dataset(). To produce an output based on the student list, use StudentList.export_template().

Some important files:

  • key_associations.txt This file contains the associations between internal keys for the Student class and other names for those keys, which may be encountered when reading a dataset. The entries are case insensitive. Any key provided that is not specified in that file will be silently ignored. You may edit this file to add new keys or associations. The format is always: key: association1, association2, etc.
  • placement_instructions.txt This file dictates how placement is to be made. See The placement_instructions.txt file for a detailed description.
